Overview

This app is designed to help you manage your home budget, on the assumption that all 
income (salary, pension etc) and expense (all outgoings) are transactions within one bank account.

The app allows for income and expense on monthly and 4-weekly intervals.
You can also add irregular income and expense items (one off items that will always come up).

The top part of the screen has opening balance (the balance of your account when you begin using the app) 
and the start date (the start date when you begin using the app)

Next are listed all your income and expense transactions, this is where you define whether they are monthly 
or 4-weekly, you should also enter the type of category (Household, Savings, Insurance, Car etc).

The bottom table is 24-month projection. When you add a transaction, the system uses the start date and 
opening balance and immediately populates the 24-month projection table with the transactions for 24 months 
from the currently defined start date.
Going forward, you can leave the start date as it is when you initially set the system up, 
but more likely you will amend the start date and opening balance, perhaps on a weekly basis, 
because once you are satisfied the transaction has come off your bank account, you have a record of that 
in that bank account. 

This app is about managing your finances going forward, and the 24-month projection table will show quite 
clearly what your finances will,be at any point in the next 24 months.

How to set up manually

First, enter a start date by clicking into Start date field and selecting a date from the date picker.
Next, enter the Opening Balance of your bank account on that date, then click Save.

You should now add your Categories. eg. Household, Insurance, Savings etc. Enter your first Category in 
the box below Start Date, then click Add Category. Add as many Categories as you need, 
you can add more as required. 
Now start adding your transactions by adding a calendar month's to ensure you 
have all your data entered.

In the box below Select Category, type in the Description, next the Amount, then select whether the 
transaction is Expense or Income. 

The next field allows to select the transaction Period
normally either Monthly or, if it's relevant 4-Weekly. 
You can select Irregular for one off transactions. 

The next field allows you to select the Date for the transaction - either the date for 
Monthly transactions, or the first date after the Start date of a 4-weekly transaction
Next select the appropriate Category for the transaction. 

Finally, click Add Transaction.
The transaction will appear in the table below. 
Add as many transactions as you require.

As you add transactions, they will appear in the 24 Month Transactiontable.

Note!
On first use, you should enter, starting from your  Start date all of your Monthly transactions 
for one calendar month. If you have 4-Weekly transactions add the next date after the start date they 
are due and ensure you enter all of this type.

The system will calculate the transactions and populate the 24 Month Projection table.

If you do have 4-Weekly transactions, once the date you enter initially has passed, the system will 
alter the day of the month shown in the transaction table by adding 28 days to ensure 
these days are always accurate

How to import transactions via CSV

Building the system (app) using Import transactions (csv) is the quickest method of creating
transactions so that the 24 Month Projection can be populated.
It relies on you creating a spreadsheet containing a row for each transaction. The spreadsheet
should then be saved as a CSV file to a location on your Computer or Tablet that is readily
accessible for any subsequent re-loads.
The format of the spreadsheet is strict and must adhere to the following specification;

The data should be in the format;
Column A:Header text=Date, content - standard date format
Column B:Header text=Amount, content - numeric
Column C:Header text=Income/Expense, content - either the text income or expense
Column D:Header text=Category , content - Text containing your chosen category
Column E:Header text=Description , content - Text containing a description of the transaction.
Column F:Header text=Frequency , content - either the text Monthly or 4-Weekly
Note! 
The header row is case sensitive and must be as shown.
The Income/Expense data is also case sensitive, lower case.
The frequency is case sensitive and must be as shown.

On the first build of the system, enter start date and opening balance, the click on 
choose file next to Import transactions CSV  then click Import (replace existing)
The transactions will instantly load. If there are any errors and alert will inform you.
Simply scroll to see all your transactions.
An example of loading data
If you have a start date of 12th Jan 2026 with an opening balance of £x.yz
your spreadsheet should have, as it's initial transaction, one dated 12th Jan 2026 (if you have one then)
ORone for the NEXTdate after that. Then add your transactions in date order from then till
11th Feb 2026. This gives you one month's worth of transactions to load into the app.
